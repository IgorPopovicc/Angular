Komponente

Komponente su glavni gradivni elementi za Angular aplikacije. Svaka komponenta se sastoji od:

  • HTML templejta koji deklariše šta se prikazuje na stranici
  • TypeScript klase koja definiše ponašanje
  • CSS selektora koji određuje kako se komponenta koristi u šablonu (templates)
  • Opciono, CSS stilova primenjenih na šablonu (templates)


Kreiranje komponente u vašoj aplikaciji može se izvršiti na dva načina:

  • Ručno putem kreiranja novih datoteka i ručnog dodavanja potrebnog koda za HTML predložak, TypeScript klasu i CSS stilove.
  • Korišćenjem Angular-ovog CLI alata pomoću komande ng generate component ime-komponente, što automatski generiše sve potrebne datoteke za novu komponentu sa osnovnom strukturom.


Kako kreirati komponentu ručno?

Iako je Angular CLI najjednostavniji način za kreiranje Angular komponenti, takođe možete ručno napraviti komponentu. Ova sekcija opisuje kako kreirati osnovnu datoteku komponente unutar postojećeg Angular projekta.


Da biste ručno kreirali novu komponentu:

  • Otvorite direktorijum vašeg Angular projekta.
  • Kreirajte novu datoteku sa nazivom <ime-komponente>.component.ts.
  • Na vrhu datoteke, dodajte sledeći import statement.
  • Nakon import statementa, dodajte @Component dekorator.
  • Izaberite CSS selektor za komponentu.
  • Izaberite stilove za templejt komponente. U većini slučajeva, definisaćete stilove za templejt komponente u posebnoj datoteci.
  • Dodajte deklaraciju klase koja uključuje kod za komponentu.

Nakon sprovođenja gore navedenih koraka, Angular komponenta će biti uspješno kreirana i spremna za upotrebu u vašoj aplikaciji.


Kako kreirati Angular komponentu koristeći Angular CLI?

Ali takođe, Angular CLI ima u sebi ugrađene komande koje sve ove postupke izvršavaju automatski, čime značajno olakšavaju i ubrzavaju proces razvoja. Da biste iskoristili ove mogućnosti, sledite sledeće korake:

  • Otvorite terminal ili command prompt.
  • Kreirajte novi Angular projekat. U terminalu, odaberite direktorijum u koji želite da se kreira projekat i unesite komandu. Zamijenite naziv-projekta sa imenom koje želite za vaš projekat:
  • Pozicionirajte se unutar direktorijuma projekta. U terminalu, unesite (Gdje je naziv-projekta ime direktorijuma projekta koje ste odabrali prilikom kreiranja):
  • Kreirajte novu komponentu koristeći Angular CLI. Zamijenite ime-komponente sa željenim imenom nove komponente:
  • Proverite strukturu direktorijuma. Otvorite projekat u vašem omiljenom tekst editoru. Struktura direktorijuma će sadržati novu komponentu unutar src/app.

Korišćenjem Angular CLI, ovi koraci automatski generišu osnovne datoteke potrebne za komponentu, uključujući HTML templejt, TypeScript klasu i CSS datoteku. Ovo ubrzava proces kreiranja komponenti i održava konzistentnost u strukturi projekta.


Životni ciklus komponente

Takođe, veoma je bitno naglasiti životni ciklus komponente. Instanca komponente ima svoj životni ciklus koji počinje kada Angular instancira klasu komponente i prikazuje njen prikaz zajedno sa pod-prikazima. Životni ciklus se nastavlja kroz proces detekcije promena, gde Angular prati kada se svojstva povezana sa podacima promene, ažurirajući i prikaz i instancu komponente po potrebi. Životni ciklus se završava kada Angular uništi instancu komponente i ukloni njen šablon iz Document Object Model (DOM). Direktive prate sličan životni ciklus, jer Angular kreira, ažurira i uništava njihove instance tokom izvršavanja.


Vaša aplikacija može koristiti metode kuke za životni ciklus (lifecycle hook methods) kako bi se uključila u ključne događaje u životnom ciklusu komponente ili direktive, omogućavajući inicijalizaciju novih instanci, pokretanje detekcije promena kada je potrebno, reagovanje na ažuriranja tokom detekcije promena i čišćenje pre nego što se instance uklone.
